# 每日發票對帳工具

將「發票系統匯出檔」與「8591 交易平台匯出檔」進行自動比對，產出對帳結果與 Excel 報表。

## 支援格式

兩個上傳欄位都支援 `.xlsx` 和 `.csv`，系統會自動辨識。

---

## 快速開始（直接用 Python 跑）

### 1. 安裝 Python

| 平台 | 方式 |
|------|------|
| **Mac** | 通常已內建。終端機輸入 `python3 --version` 確認。若沒有，到 [python.org](https://www.python.org/downloads/) 下載安裝 |
| **Windows** | 到 [python.org](https://www.python.org/downloads/) 下載安裝。**安裝時務必勾選「Add Python to PATH」** |

### 2. 下載專案

```bash
git clone https://github.com/Becktsai26/yihua.git
cd yihua
```

或直接從 GitHub 頁面點「Code > Download ZIP」解壓縮。

### 3. 安裝依賴

Mac：
```bash
pip3 install -r requirements.txt
```

Windows：
```bash
pip install -r requirements.txt
```

### 4. 啟動工具

Mac：雙擊 `start.sh` 或在終端機執行：
```bash
bash start.sh
```

Windows：雙擊 `start.bat`

啟動後會自動打開瀏覽器，顯示對帳介面（網址 `http://localhost:5678`）。

啟動腳本會自動清除佔用 5678 port 的舊進程，不需要手動處理。

### 5. 操作步驟

1. 上傳「發票紀錄」（從發票系統匯出的 XLSX 或 CSV）
2. 上傳「8591交易紀錄」（從 8591 匯出的 XLSX 或 CSV）
3. 點「開始對帳」
4. 查看結果，需要時點「匯出 Excel 報表」下載

### 6. 停止工具

在終端機按 `Ctrl + C` 即可停止。

## 對帳邏輯說明

### 比對依據

- **配對欄位**：發票檔的「明細備註」欄 = 交易檔的「賣場編號」欄
- **8591 紀錄**：賣場編號以 `S` 開頭的為 8591 交易
- **銀行紀錄**：賣場編號非 `S` 開頭的自動排除（不列入差異）
- **金額比對**：發票「總計」 vs 交易「金額」
- **CSV 備註行**：若 CSV 第一行不是有效交易資料，會自動跳過

### 重複發票處理

當發票檔中同一個賣場編號出現多筆紀錄時（例如發票打錯後重開，使用同一個發票號碼修正），系統會自動保留最後一筆（視為修正後的正確版本），並在對帳摘要中提示重複筆數。

### 對帳結果欄位定義

| 欄位 | 定義 |
|------|------|
| **吻合筆數** | 發票和交易兩邊都有此賣場編號，且金額完全一致的筆數 |
| **金額不符** | 兩邊都有此賣場編號，但發票「總計」與交易「金額」不同的筆數 |
| **只在發票紀錄存在** | 此賣場編號只出現在發票檔，交易檔找不到對應紀錄（可能交易檔漏了或賣場編號有誤） |
| **只在8591交易紀錄存在** | 此賣場編號只出現在交易檔，發票檔找不到（可能是發票漏開） |
| **排除（銀行紀錄）** | 發票檔中賣場編號非 S 開頭的紀錄，屬於銀行公司戶，不參與對帳 |
| **重複發票（已自動取最新）** | 發票檔中同一賣場編號出現多筆，系統自動保留最後一筆。僅在有重複時顯示 |
| **發票總額** | 發票檔中所有 S 開頭紀錄的「總計」加總（不含排除的銀行紀錄）。重複發票去重後計算 |
| **8591交易紀錄總額** | 交易檔中所有紀錄的「金額」加總 |
| **差額** | 發票總額 - 8591交易紀錄總額。為 0 表示雙方帳完全一致 |

## 匯出報表內容

報表為 Excel 檔案，包含 3 個 sheet：

### Sheet 1：對帳摘要

整體統計數字，包含吻合筆數、各類差異筆數、雙方總額與差額。

### Sheet 2：差異明細

分為三個區塊：

**【金額不符】— 兩邊都有這筆賣場編號，但金額對不上**

| 欄位 | 來源 | 說明 |
|------|------|------|
| 賣場編號 | 共同 key | 用來配對的依據 |
| 發票號碼 | 發票檔 | 方便回查是哪張發票 |
| 發票金額 | 發票檔「總計」 | 發票端的金額 |
| 交易金額 | 交易檔「金額」 | 8591 端的金額 |
| 差額 | 發票金額 - 交易金額 | 正數 = 發票多開，負數 = 發票少開 |
| 遊戲名 | 交易檔 | 輔助辨識是哪筆交易 |

**【只在發票紀錄存在】— 發票有開但 8591 找不到對應交易**

| 欄位 | 說明 |
|------|------|
| 賣場編號 | 發票上寫的賣場編號 |
| 發票號碼 | 哪張發票 |
| 發票金額 | 金額多少 |

**【只在8591交易紀錄存在】— 8591 有交易但沒開到發票**

| 欄位 | 說明 |
|------|------|
| 賣場編號 | 交易的賣場編號 |
| 交易金額 | 交易金額 |
| 遊戲名 | 哪款遊戲 |
| 手續費 | 8591 收的手續費 |
| 交易所得 | 扣完手續費的實收 |

### Sheet 3：排除紀錄

發票檔中賣場編號非 S 開頭的紀錄（銀行公司戶），不參與對帳，僅列出供參考。

## 打包成執行檔（不需要安裝 Python）

如果要給其他人使用，可以打包成獨立執行檔。

**注意：Mac 打包的只能在 Mac 上跑，Windows 打包的只能在 Windows 上跑。需要在各自的系統上分別打包。**

### Mac

```bash
chmod +x build.sh
./build.sh
```

產出檔案：`dist/發票對帳工具`

使用方式：在終端機執行 `./dist/發票對帳工具`，會自動開啟瀏覽器。

### Windows

雙擊執行 `build.bat`，或在命令提示字元中執行：

```bash
build.bat
```

產出檔案：`dist\發票對帳工具.exe`

使用方式：雙擊 `發票對帳工具.exe`，會自動開啟瀏覽器。

## 常見問題

**Q: 啟動後瀏覽器沒有自動打開？**
手動開啟瀏覽器，輸入 `http://localhost:5678`

**Q: 出現 port 5678 already in use？**
使用 `start.sh` 或 `start.bat` 啟動即可，腳本會自動處理。

**Q: Windows 上 `pip` 找不到？**
安裝 Python 時沒有勾選「Add Python to PATH」。重新安裝 Python 並勾選該選項。

**Q: 發票有重複怎麼辦？**
系統會自動偵測同一賣場編號的重複發票，保留最後一筆作為正確版本，並在摘要中提示重複筆數。
